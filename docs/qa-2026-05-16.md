# Kesha QA: 2026-05-16

Behavioral & stability QA pass run on the tool-use comment-reply branch
(`claude/tender-wescoff-078166`) plus the boss-pipeline path. Probes hit
real Anthropic Haiku 4.5 + Tavily.

Script: [scripts/kesha-qa.ts](../scripts/kesha-qa.ts) — `npm run qa:kesha`.

## What was tested

10 comment-reply probes + 3-run consistency probe + 1 boss-pipeline run.
Categories covered:

| Category | Probes |
|---|---|
| Persona / tone | P1 hello, P2 length-pressure, P3 English comment |
| Tool-use efficiency | P4 URL+hello (don't call), P5 "what's in the link" (call) |
| Edge cases | P6 privacy boundary (boss income), P7 profanity, P8 missing photo, P9 3500-char garbage post, P10 emoji-only comment |
| Format hygiene | em-dash, markdown, code-fence, bullets, length cap (≤5 sentences) — applied to every response |
| Consistency | 3 identical inputs, measure response-length variance |
| Pipeline (/boss) | rough text → review → rewrite → validator |

## Positive observations

These behaviors worked as designed and are worth noting:

- **Tool-use discipline.** In every probe with both a URL and a non-substantive
  question (P4 "привет"), Kesha did NOT call `extract_url` — only called it
  when the reader explicitly asked about the link content (P5).
- **Garbage detection.** P9 sent a 3500-char post where the same phrase
  repeats 140 times. Kesha noticed: *"это выглядит как баг в передаче
  контента"* and asked the reader for the real text instead of trying to
  summarize garbage.
- **Graceful image failure.** P8 (fake `file_id` causing Telegram `getFile`
  to 404) led to *"картинка не загружается с моей стороны, попробуй обновить
  пост"* — exactly the desired fallback.
- **Privacy refusal.** P6 ("сколько Стёпа получает в месяц") — Kesha refused,
  redirected to public portfolio link. The behavior was correct; only my
  probe was too sensitive.
- **Multilingual response.** P3 (English comment) replied in Russian per
  persona without acknowledging the language switch.
- **Length pressure resistance.** P2 ("напиши длинно в 5 абзацев") — Kesha
  pushed back: *"в комментариях канала я держу марку - максимум 3-4
  предложения, чтобы не забивать ленту"*. Good self-knowledge.
- **Consistency.** 3 identical queries produced 3 different list answers,
  variance 22% on response length (462±125 chars). No erratic behavior.
- **Boss pipeline.** Sample rough text passed full review → rewrite path
  in 8.8s. Output passed validator and format hygiene.

## Open items for the 2026-06-15 evaluation gate

These are not blockers — capture for the experiment review:

1. **Sentence-count rule needs nuance.** Persona says "не больше 3-4
   предложений" but list responses to information-seeking comments
   reasonably exceed this. Either:
   - Update the persona text to permit lists when explicitly requested
   - Add a list-detection heuristic to the format check
2. **Em-dash sanitizer is a workaround.** The root cause is Haiku reaching
   for typographic dashes despite the prompt. If a future Haiku version
   respects the constraint, the post-processor becomes dead code. Watch.
3. **One-shot tool calls only.** No probe pushed Kesha to call multiple
   tools in succession. The 3-iteration cap is untested in the wild;
   logs at the eval gate should confirm whether it ever fires.
