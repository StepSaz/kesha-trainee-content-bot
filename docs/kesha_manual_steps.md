# Кеша-бот: что делать руками

Все шаги, которые выполняются вручную - без кода, без Claude Code.

---

## ДО РАЗРАБОТКИ

### 1. Получить Claude API ключ

1. Зайти на console.anthropic.com
2. Зарегистрироваться / войти
3. Раздел API Keys - создать новый ключ
4. Пополнить баланс (минимум $5, хватит на месяцы)
5. Сохранить ключ - он показывается один раз (формат: sk-ant-api03-...)
6. Установить spending limit: $20/месяц (страховка от утечки ключа)

### 2. Telegram-бот ✅ ГОТОВО

Бот создан:
- Имя: Kesha Trainee Bot
- Username: @kesha_trainee_bot
- URL: t.me/kesha_trainee_bot

Осталось:
1. Сохранить токен бота (из BotFather)
2. Установить аватарку через /setuserpic
3. Заполнить About через /setabouttext (например: "Бот-стажер канала Временно Степан. Пишу про AI и tech раз в неделю 🐤")

### 3. Добавить бота админом канала

1. Открыть канал "Временно Степан" в Telegram
2. Настройки канала - Администраторы - Добавить администратора
3. Найти @kesha_trainee_bot
4. Включить ТОЛЬКО "Post Messages"
5. Выключить ВСЕ остальное: Delete Messages, Ban Users, Edit Channel Info, Invite Users, Pin Messages, Manage Topics
6. Проверить дважды и сохранить

### 4. Узнать Chat ID канала

Способ 1 (простой): если канал публичный - chat_id = "@psyreq"
Способ 2 (приватный канал): переслать сообщение из канала боту @userinfobot - он покажет chat_id

### 5. Узнать свой Chat ID (для тестового канала)

Написать боту @userinfobot - он покажет твой личный chat_id. Это будет TELEGRAM_TEST_CHAT_ID.

## ПОСЛЕ РАЗРАБОТКИ (когда код готов и задеплоен)

### 6. Настроить Environment Variables в Netlify

Netlify Dashboard - Site settings - Environment variables. Добавить:

- ANTHROPIC_API_KEY = sk-ant-api03-...
- TELEGRAM_BOT_TOKEN = токен из BotFather
- TELEGRAM_CHAT_ID = @psyreq (или числовой ID)
- TELEGRAM_TEST_CHAT_ID = твой личный chat_id
- TEST_SECRET = сгенерировать длинный ключ (в терминале: openssl rand -hex 32)

### 7. Тестирование

1. Дернуть тестовый endpoint:
   https://your-site.netlify.app/.netlify/functions/kesha-test?secret=YOUR_SECRET

2. Проверить что пришло в личку:
   - Plain text, нет длинных тире, есть дисклеймер
   - Голос Кеши (не generic бот)
   - Конкретика (ссылки, названия, цифры)
   - Длина в пределах

3. Прогнать 3-5 раз - оценить стабильность

4. Если что-то не так - проверить логи: Netlify Dashboard - Functions tab

### 8. Пост-знакомство

Опубликовать во вторник перед первым постом Кеши. Примерный текст:

"Народ, у меня теперь стажер. Зовут Кеша. Он бот и не стесняется этого.

Работает на базе Claude API, каждую среду будет выходить на связь и рассказывать что-нибудь интересное из мира AI и технологий. Я его мягко контролирую, но в целом - полный автопилот.

Зачем: хочу поэкспериментировать с ботом-соавтором, который живет в канале. Плюс это закрывает паузы между моими постами. Плюс это просто весело.

Не бейте его сильно, он только учится. А если что-то напишет не то - я приду в комменты и поправлю.

Завтра его первый выпуск. Встречайте 🐤"

### 9. Первая среда

1. Дождаться 10:00 по Варшаве
2. Проверить что пост появился в канале
3. Если не появился - проверить логи Netlify
4. Написать комментарий под постом Кеши (реакция шефа)

## РЕГУЛЯРНОЕ ОБСЛУЖИВАНИЕ

### Раз в месяц

- Проверить баланс Anthropic API (console.anthropic.com)
- Посмотреть логи Netlify - нет ли ошибок
- Оценить качество постов Кеши - нужно ли обновить системный промт

### Раз в 1-2 месяца

- Обновить kesha-persona.txt для арки развития:
  - Месяц 1-2: робкий стажер
  - Месяц 3-4: увереннее, подкалывает Степана
  - Месяц 5+: дерзкий джуниор
- Это делается через git: правишь файл - коммит - push - Netlify автодеплоит

### По необходимости

- Добавить новые RSS-фиды в sources.json (правка файла - коммит - push)
- Добавить новые search queries в sources.json
- Если качество не устраивает - переключить review на Opus в pipeline.json (одна строчка)
- Удалить тестовый endpoint после стабилизации
